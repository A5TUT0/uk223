MiniTwitter Task Checklist

  • Diseño del esquema de base de datos (ERM)

    Alta prioridad

    Crea un modelo ER completo utilizando la notación de pata de cuervo que refleje el esquema de la base de datos. Incluir:

    • Entidades clave como Usuarios, Publicaciones, Comentarios y Roles.
    • Relaciones entre estas entidades con cardinalidades claras.

  • Diagrama UML de la lógica del backend

    Alta prioridad

    Elabora un diagrama UML que cubra:

    • Clases principales como Usuario, Publicación, Comentario y Rol.
    • Sus métodos, atributos y relaciones.
    • Modificadores de acceso (private, protected, public).

  • Registro e inicio de sesión

    Alta prioridad

    Desarrolla las funcionalidades para:

    • Registro de usuarios con nombre de usuario y contraseña.
    • Inicio de sesión seguro, validando credenciales en el backend.
    • Implementar cifrado seguro de contraseñas (por ejemplo, bcrypt).

  • Gestión de publicaciones

    Alta prioridad

    Implementa las siguientes funcionalidades:

    • Crear publicaciones con texto limitado (similar a tweets).
    • Editar publicaciones propias.
    • Eliminar publicaciones.

  • Comentarios en publicaciones

    Media prioridad

    Asegúrate de que los usuarios puedan:

    • Agregar comentarios a publicaciones existentes.
    • Editar sus propios comentarios.
    • Eliminar comentarios propios.

  • Gestión de roles

    Media prioridad

    Implementa los siguientes roles y permisos:

    • Usuario: Crear, editar y eliminar sus propias publicaciones y comentarios.
    • Moderador: Editar y eliminar cualquier publicación o comentario.
    • Administrador: Gestión completa de usuarios, publicaciones y comentarios (incluido bloquear usuarios).

  • Funcionalidad de Me gusta / No me gusta

    Baja prioridad

    Permitir que los usuarios puedan:

    • Valorar publicaciones con "Me gusta" o "No me gusta".
    • Visualizar el conteo de estas interacciones en cada publicación.

  • Pruebas y documentación

    Baja prioridad

    Incluye:

    • Definición de casos de prueba que cubran todas las funcionalidades principales.
    • Un informe detallado de los resultados de las pruebas.
    • Documentación técnica sobre la arquitectura y el uso del sistema.

  • Frontend y diseño

    Baja prioridad

    Usa las mejores prácticas para el diseño:

    • Crear una interfaz de usuario amigable y responsiva.
    • Organizar el código del frontend en módulos y archivos coherentes.
    • Utilizar frameworks CSS como Bootstrap o Tailwind si es necesario.
